# Go-to-market & Traction Plan

## 1. Initial wedge

Start with Vietnamese beginner English learners who already encounter English in daily life and need Vietnamese explanations.

Good early user groups:

| Segment | Why it fits |
|---|---|
| Students | Frequent English exposure, need simple explanations |
| Office workers | Emails/messages/signs create real use cases |
| Self-learners | Actively searching for tools |
| Travelers | Clear scan-and-understand use cases |
| English learning communities | Easy to recruit beta testers |

---

## 2. Positioning

Main message:

```text
Snap a piece of English and turn it into an easy-to-understand lesson in Vietnamese.
```

Avoid positioning as:

- generic translator
- full English course
- perfect grammar authority
- replacement for teacher

Position as:

```text
Learning assistant for real-world English.
```

---

## 3. Beta acquisition channels

| Channel | Test |
|---|---|
| Facebook groups | Post demo examples: menu, flyer, email |
| TikTok/Reels | Short before/after scan-to-lesson videos |
| Zalo/community groups | Recruit closed beta testers |
| Student groups | Test actual learning usefulness |
| English teachers/tutors | Get feedback on lesson quality |
| App store listing | Later, once beta signals are stable |

---

## 4. 30/60/90-day traction plan

| Period | Objective | Success signals |
|---|---|---|
| First 30 days | Internal beta and quality validation | 20+ sample images pass, schema valid rate stable, no core crash |
| Days 31-60 | Closed beta with real users | 100-300 testers, generated lesson count, save rate, qualitative feedback |
| Days 61-90 | Retention and monetization signal | D7 retention, review saved lesson rate, BYOK setup success, paid intent |

---

## 5. Metrics to report

| Metric | Meaning |
|---|---|
| Activated users | Users who generate at least 1 lesson |
| Lessons generated | Core usage volume |
| Scan vs paste ratio | Validates scan use case |
| OCR success rate | Input quality |
| AI valid output rate | Reliability |
| Save rate | User sees lesson value |
| Review rate | Retention signal |
| Cost per lesson | Unit economics |
| BYOK adoption | Cost reduction potential |
| Paid intent | Monetization signal |

---

## 6. Early experiments

| Experiment | Question answered |
|---|---|
| Scan flyer/menu/email demos | Which use case resonates most? |
| Paste text only prototype | Is learning output valuable without OCR? |
| BYOK setup | Can free/power users configure their own AI key? |
| Trial quota | How many free lessons are enough for activation? |
| Save/review | Does user return to saved lesson? |
| Teacher feedback | Is explanation quality acceptable for beginners? |

---

## 7. What traction would make the idea stronger

Strong early signals:

- Users generate multiple lessons from real-world content.
- Users save lessons and reopen them.
- Users say explanations are easier than generic translation.
- OCR and AI errors do not break trust.
- Cost per lesson stays within guardrails.
- Some users show willingness to pay for managed AI/no setup.
